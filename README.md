# Claude Code Remote

A voice-only iOS app for hands-free, eyes-free access to Claude Code.  
Your friend speaks → Claude Code thinks → the response is read aloud.

```
iPhone mic → iOS app → bot server (your Mac) → claude CLI → response → iOS TTS
                     ↑                                               ↓
               (HTTP over ngrok)                        Telegram push notification
                                                         (if app is backgrounded)
```

## Part 1 — Bot server setup (your Mac)

### Prerequisites
- **Node.js** — for `claude` CLI: https://nodejs.org
- **Python 3.11+**
- **ngrok** — to expose the local server:
  ```bash
  brew install ngrok/ngrok/ngrok
  ngrok config add-authtoken <your-token>   # dashboard.ngrok.com/get-started/your-authtoken
  ```

### Steps

```bash
# 1. Install and log in to Claude Code
npm install -g @anthropic-ai/claude-code
claude login

# 2. Run the setup script
cd bot/
bash setup.sh

# 3. Fill in your Telegram bot token (optional but great for push notifications)
#    Create a bot at https://t.me/BotFather, copy the token, paste into bot/.env
nano .env

# 4. Start the server
source .venv/bin/activate
python server.py
# → Listening on http://0.0.0.0:8080

# 5. In a second terminal, expose it to the internet
ngrok http 8080
# → Forwarding  https://xxxx-xx-xx-xxx-xx.ngrok-free.app -> http://localhost:8080
#              ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
#              Copy this URL — you'll paste it into the iOS app settings
```

The server stays running while your friend uses the app. You can keep a terminal open or run it with `nohup python server.py &`.

### Telegram push notifications (optional)

1. Open Telegram and message **@BotFather** → `/newbot` → follow prompts → copy the token.
2. Add it to `bot/.env` as `TELEGRAM_BOT_TOKEN=<token>`.
3. Start (or restart) the server.
4. Have your friend open Telegram and message the new bot — it will auto-confirm the connection.  
   From then on, every Claude Code response is also delivered as a Telegram message for when the iOS app is closed.

## Part 2 — iOS app setup

### Requirements
- Xcode 15+
- An Apple Developer account (free is fine for personal device testing)
- iPhone running iOS 16+

### Build and install

```bash
# From the repo root — regenerate the Xcode project if you've added/renamed files
xcodegen generate

# Open in Xcode
open ClaudeCodeRemote.xcodeproj
```

In Xcode:
1. Select the `ClaudeCodeRemote` target → **Signing & Capabilities** → choose your Team.
2. Plug in the iPhone → select it as the run destination → **Product → Run** (⌘R).

### Configure the server URL

The first time (or after ngrok gives a new URL):

1. Open the app on the iPhone.
2. **Long press** anywhere on the screen for 1.5 seconds → Settings opens.
3. Paste the ngrok `https://` URL into the **Bot Server URL** field → **Save**.

That's all the configuration your blind friend ever needs to do (or have done for them once).

## How the app works (user's perspective)

| What happens | What the user hears |
|---|---|
| App opens | "Welcome to Claude Code Remote. Say **new session** to start fresh, or say **continue** to pick up where you left off." |
| User says "new session" | "Starting a new session. What would you like to work on?" |
| User describes their coding task | "Got it. Sending to Claude Code now." |
| Claude Code responds | The full response is read aloud. |
| Response finished | "Tap anywhere to respond." |
| User taps → speaks follow-up | Repeat from send step |
| App is closed, response arrives | Telegram push notification |
| User taps notification → app opens | Response is read aloud automatically |

**Tap during speaking** — stops the current speech and immediately starts listening.  
**Long press** — opens Settings (for sighted setup).

## Troubleshooting

| Symptom | Fix |
|---|---|
| "Cannot reach the server" | Make sure `python server.py` is running and ngrok is active. Re-paste the URL in Settings if ngrok restarted. |
| "Permissions required" | Go to iPhone Settings → Claude Code Remote → enable Microphone and Speech Recognition. |
| Claude Code times out | Reduce the complexity of the prompt, or increase `TIMEOUT` in `claude_runner.py`. |
| Telegram notifications not arriving | Confirm `TELEGRAM_BOT_TOKEN` is set in `.env` and the server was restarted after editing. Send `/start` to the bot on Telegram. |
| Xcode signing error | Select a Development Team in Signing & Capabilities. A free Apple ID works for personal use. |
